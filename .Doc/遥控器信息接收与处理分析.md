# 遥控器信息接收与处理框架分析

## 概述

框架提供了一套完整的遥控器（SBUS协议）信息接收和处理系统，位于 `modules/remote/` 目录。遥控器数据通过USART3接收，使用DMA+中断方式，支持离线检测。

---

## 一、核心文件清单

| 文件 | 说明 |
|------|------|
| `modules/remote/remote_control.c` | 遥控器接收与解析实现 |
| `modules/remote/remote_control.h` | 遥控器数据结构与API定义 |
| `modules/remote/remote.md` | 遥控器硬件通道映射说明 |
| `application/cmd/robot_cmd.c` | 遥控器数据应用处理示例 |
| `bsp/usart/bsp_usart.c/h` | 串口BSP层封装 |

## 二、数据结构详解

### 2.1 RC_ctrl_t（遥控器数据结构）

```c
typedef struct {
    struct {
        int16_t rocker_l_;     // 左摇杆水平 (通道0)
        int16_t rocker_l1;      // 左摇杆竖直 (通道1)
        int16_t rocker_r_;      // 右摇杆水平 (通道2)
        int16_t rocker_r1;      // 右摇杆竖直 (通道3)
        int16_t dial;           // 左侧拨轮 (通道4)
        uint8_t switch_left;    // 左侧开关
        uint8_t switch_right;   // 右侧开关
    } rc;
    struct {
        int16_t x;              // 鼠标X轴
        int16_t y;              // 鼠标Y轴
        uint8_t press_l;        // 鼠标左键
        uint8_t press_r;        // 鼠标右键
    } mouse;
    Key_t key[3];               // 键盘状态 [0]=按下, [1]=保持, [2]=Ctrl组合, [3]=Shift组合
    uint8_t key_count[3][16];   // 按键计数,用于检测单次按下
} RC_ctrl_t;
```

### 2.2 Key_t（键盘位域结构）

```c
typedef union {
    struct {
        uint16_t w : 1;         // Bit 0
        uint16_t s : 1;         // Bit 1
        uint16_t d : 1;         // Bit 2
        uint16_t a : 1;         // Bit 3
        uint16_t shift : 1;     // Bit 4
        uint16_t ctrl : 1;      // Bit 5
        uint16_t q : 1;         // Bit 6
        uint16_t e : 1;         // Bit 7
        uint16_t r : 1;         // Bit 8
        uint16_t f : 1;         // Bit 9
        uint16_t g : 1;         // Bit 10
        uint16_t z : 1;         // Bit 11
        uint16_t x : 1;         // Bit 12
        uint16_t c : 1;         // Bit 13
        uint16_t v : 1;         // Bit 14
        uint16_t b : 1;         // Bit 15
    };
    uint16_t keys;              // 用于memcpy
} Key_t;
```

### 2.3 通道值范围

| 常量 | 值 | 说明 |
|------|-----|------|
| `RC_CH_VALUE_MIN` | 364 | 通道最小值 |
| `RC_CH_VALUE_MAX` | 1684 | 通道最大值 |
| `RC_CH_VALUE_OFFSET` | 1024 | 通道中间值（偏置） |

摇杆值范围: **-660 ~ +660**（中间值1024，偏置后为0）

### 2.4 开关状态定义

| 宏 | 值 | 物理位置 |
|-----|-----|----------|
| `RC_SW_UP` | 1 | 开关向上 |
| `RC_SW_MID` | 3 | 开关中间 |
| `RC_SW_DOWN` | 2 | 开关向下 |

## 三、API函数说明

### 3.1 RemoteControlInit()

**函数原型:**
```c
RC_ctrl_t *RemoteControlInit(UART_HandleTypeDef *rc_usart_handle);
```

**参数:**
- `rc_usart_handle`: 遥控器对应的UART句柄（DT7使用USART3）

**返回值:**
- `RC_ctrl_t *`: 遥控器数据指针

**功能:**
- 注册串口接收回调
- 注册Daemon守护进程（100ms超时检测离线）
- 初始化遥控器数据缓冲区

**使用示例:**
```c
rc_data = RemoteControlInit(&huart3);
```

---

### 3.2 RemoteControlIsOnline()

**函数原型:**
```c
uint8_t RemoteControlIsOnline();
```

**返回值:**
- `1`: 在线
- `0`: 离线或未初始化

**功能:**
- 检查遥控器是否在线
- 内部调用DaemonIsOnline()

## 四、内部函数说明

### 4.1 sbus_to_rc()

**功能:** SBUS协议解析，将18字节原始数据解析为遥控器数据结构

**处理内容:**
1. 摇杆通道解析（4个通道 + 拨轮）
2. 开关状态解析
3. 鼠标数据解析
4. 键盘数据解析（支持Ctrl/Shift组合键）
5. 按键计数更新（检测上升沿）

### 4.2 RectifyRCjoystick()

**功能:** 矫正摇杆值

**规则:** 超过660或小于-660的值被视为无效，置0

### 5.1 内部静态变量

| 变量 | 类型 | 作用 |
|------|------|------|
| `rc_ctrl[2]` | RC_ctrl_t[2] | [TEMP]=当前数据, [LAST]=上一次数据 |

| `rc_daemon_instance` | DaemonInstance* | 守护进程实例指针 |

### 5.2 变量访问模式

```c
// 获取当前数据
rc_data[TEMP].rc.rocker_l_;    // 左摇杆水平
rc_data[TEMP].mouse.x;         // 鼠标X
rc_data[TEMP].key[KEY_PRESS].w; // W键是否按下

// 获取上一次数据
rc_data[LAST].rc.rocker_l_;    // 上一次左摇杆值

// 获取按键计数（用于检测单次按下）
rc_data[TEMP].key_count[KEY_PRESS][Key_W]; // W键被按下的次数
```

## 六、使用方法（让遥控器起作用）

### 6.1 初始化步骤

**Step 1:** 在 `RobotCMDInit()` 中调用初始化
```c
// application/cmd/robot_cmd.c
void RobotCMDInit() {
    rc_data = RemoteControlInit(&huart3);  // DT7使用USART3
    // ... 其他初始化
}
```

**Step 2:** 确保USART3在CubeMX中配置正确
- 波特率: 100000
- 数据位: 8
- 停止位: 2
- 校验: None
- 硬件流控: RTS/CTS（可选）

### 6.2 读取遥控器数据

**方式一：直接读取摇杆值**
```c
// 摇杆值范围 -660 ~ +660
float vx = rc_data[TEMP].rc.rocker_r_ / 660.0f * MAX_SPEED;
float vy = rc_data[TEMP].rc.rocker_r1 / 660.0f * MAX_SPEED;
```

**方式二：读取开关状态**
```c
if (switch_is_down(rc_data[TEMP].rc.switch_right)) {
    // 右侧开关向下
} else if (switch_is_mid(rc_data[TEMP].rc.switch_right)) {
    // 右侧开关中间
} else if (switch_is_up(rc_data[TEMP].rc.switch_right)) {
    // 右侧开关向上
}
```

**方式三：读取键盘按键（单次按下检测）**
```c
// 检测W键单次按下
if (rc_data[TEMP].key_count[KEY_PRESS][Key_W] != rc_data[LAST].key_count[KEY_PRESS][Key_W]) {
    // W键被按下一次
}

// 检测Ctrl+W组合键
if (rc_data[TEMP].key[KEY_PRESS].ctrl && rc_data[TEMP].key[KEY_PRESS].w) {
    // Ctrl+W被按下
}
```

### 6.3 急停功能

**自动触发条件:**
- 拨轮向下超过300
- 重要模块离线

**代码位置:** `EmergencyHandler()` in `robot_cmd.c`

## 七、开关控制逻辑示例

### 7.1 右侧开关控制底盘/云台模式

```c
// 右侧开关[下] - 底盘跟随云台
if (switch_is_down(rc_data[TEMP].rc.switch_right)) {
    chassis_cmd_send.chassis_mode = CHASSIS_ROTATE;
    gimbal_cmd_send.gimbal_mode = GIMBAL_GYRO_MODE;
}
// 右侧开关[中] - 底盘云台分离
else if (switch_is_mid(rc_data[TEMP].rc.switch_right)) {
    chassis_cmd_send.chassis_mode = CHASSIS_NO_FOLLOW;
    gimbal_cmd_send.gimbal_mode = GIMBAL_FREE_MODE;
}
```

### 7.2 左侧开关控制输入源

```c
// 左侧开关[下] - 遥控器控制
if (switch_is_down(rc_data[TEMP].rc.switch_left)) {
    RemoteControlSet();
}
// 左侧开关[上] - 键鼠控制
else if (switch_is_up(rc_data[TEMP].rc.switch_left)) {
    MouseKeySet();
}
```

### 7.3 拨轮控制摩擦轮和发射

```c
// 拨轮向上超过100 - 开启摩擦轮
if (rc_data[TEMP].rc.dial < -100) {
    shoot_cmd_send.friction_mode = FRICTION_ON;
} else {
    shoot_cmd_send.friction_mode = FRICTION_OFF;
}

// 拨轮向上超过500 - 连发模式
if (rc_data[TEMP].rc.dial < -500) {
    shoot_cmd_send.load_mode = LOAD_BURSTFIRE;
}
```

## 八、注意事项

### 8.1 硬件连接
- DT7遥控器使用**USART3**（注意选用带反相器的串口）
- 波特率100000，2停止位，8数据位

### 8.2 数据有效性
- 摇杆值超过±660会被自动置0
- 遥控器离线后数据会被清零

### 8.3 组合键检测
- Ctrl和Shift组合键有独立的状态存储
- 按键计数只统计**上升沿**（从0到1的变化）

### 8.4 离线检测
- 100ms未收到数据视为离线
- 离线后自动调用 `RCLostCallback()` 清空数据并尝试重连

### 8.5 TEMP和LAST的使用
- `[TEMP]`: 当前数据
- `[LAST]`: 上一次数据
- 用于判断按键是否刚刚按下（单次触发）
