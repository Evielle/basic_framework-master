# 四个底盘电机验证方案（M3508×2 + LK9015×2，CAN2）

## 概述

根据电机总表，需要验证的四个电机：

| 物理位置 | 电机型号 | CAN总线 | 内部ID | CAN反馈ID | CAN控制ID |
|---|---|---|---|---|---|
| 左舵向 | DJI M3508 | CAN2 / hcan2 | 1 | 0x201 | 0x200 (多电机分组发送) |
| 右舵向 | DJI M3508 | CAN2 / hcan2 | 2 | 0x202 | 0x200 (多电机分组发送) |
| 左轮向 | LK9015 | CAN2 / hcan2 | 1 | 0x141 | 0x280 (多电机发送) |
| 右轮向 | LK9015 | CAN2 / hcan2 | 2 | 0x142 | 0x280 (多电机发送) |

本文档重点说明：
1. 框架中哪些函数可验证电机
2. 如何正确初始化这四个电机
3. 如何发送控制指令并检查反馈
4. 验证流程和注意事项

## 一、可用于验证的核心函数清单

### 1.1 DJI M3508（舵向电机）相关函数

位置：`modules/motor/DJImotor/dji_motor.h`

| 函数 | 作用 | 验证用途 |
|------|------|----------|
| `DJIMotorInit(Motor_Init_Config_s *config)` | 初始化电机,注册CAN | 验证电机是否能正常注册到CAN总线 |
| `DJIMotorSetRef(DJIMotorInstance *m, float ref)` | 设置参考输入 | 发送速度/位置/电流指令 |
| `DJIMotorControl()` | 计算PID并发送控制报文 | 控制循环,被MotorControlTask调用 |
| `DJIMotorStop(DJIMotorInstance *m)` | 停止电机 | 安全停止 |
| `DJIMotorEnable(DJIMotorInstance *m)` | 使能电机 | 使电机响应参考输入 |
| `DaemonIsOnline(DaemonInstance *d)` | 检查在线状态 | 验证CAN反馈是否正常 |

**DJI电机反馈数据读取**：
```c
DJIMotorInstance *m;

// 编码器
uint16_t ecd = m->measure.ecd;              // 原始编码器值(0~8191)
float single_angle = m->measure.angle_single_round;  // 单圈角度(0~360)
float total_angle = m->measure.total_angle; // 多圈累计角度
int32_t total_round = m->measure.total_round; // 总圈数

// 速度和电流
float speed = m->measure.speed_aps;          // 角速度(度/秒)
int16_t current = m->measure.real_current;   // 实际电流
uint8_t temp = m->measure.temperature;      // 温度
```

### 1.2 LK9015（轮向电机）相关函数

| 函数 | 作用 | 验证用途 |

## 二、CAN ID工作原理详解

### 2.1 DJI M3508的CAN机制

**初始化流程**（`DJIMotorInit()` → `MotorSenderGrouping()`）：

```
用户传入tx_id = 1或2
       ↓
MotorSenderGrouping()内部处理:
  rx_id = 0x200 + tx_id = 0x201 或 0x202   ← 【反馈报文ID】
  分组sender_group = CAN2时为4              ← 【发送分组】
  发送时使用sender_assignment[4], CAN ID=0x200
  tx_buff[0]~[1] 为ID=1电机电流
  tx_buff[2]~[3] 为ID=2电机电流
  tx_buff[4]~[7] 为ID=3,4电机电流(本项目未用)
```

**报文格式**（控制发送）：
```
CAN ID: 0x200, 数据长度: 8字节
Byte[0-1]: ID=1电机电流 (int16, 高字节在前)
Byte[2-3]: ID=2电机电流
Byte[4-5]: ID=3电机电流
Byte[6-7]: ID=4电机电流
```

**报文格式**（反馈接收）：
```
CAN ID: 0x201 (电机1), 0x202 (电机2)
Byte[0-1]: 编码器值 ecd
Byte[2-3]: 转速 rpm
Byte[4-5]: 实际电流
Byte[6]  : 温度
```

### 2.2 LK9015的CAN机制

**初始化流程**（`LKMotorInit()`）：

```
用户传入tx_id = 1或2
       ↓
LKMotorInit()内部处理:
  rx_id = 0x140 + tx_id = 0x141 或 0x142   ← 【反馈报文ID】
  tx_id被修改为 tx_id + 0x280 - 1 = 0x280 或 0x281 (实际发送仅用第一个电机的实例)
  sender_instance保存第一个电机的can_ins,其tx_id被设为0x280
  发送时: tx_buff[(motor->can_ins->tx_id - 0x280) * 2]
          = tx_buff[0]~[1] 为ID=1电机
          = tx_buff[2]~[3] 为ID=2电机
```

**报文格式**（控制发送）：
```
CAN ID: 0x280, 数据长度: 8字节
Byte[0-1]: ID=1电机电流
Byte[2-3]: ID=2电机电流
Byte[4-5]: ID=3电机电流
Byte[6-7]: ID=4电机电流
```

**报文格式**（反馈接收）：
```
CAN ID: 0x141 (电机1), 0x142 (电机2)
Byte[0]  : 状态/帧标识
Byte[1]  : 温度
Byte[2-3]: 实际电流
Byte[4-5]: 转速
Byte[6-7]: 编码器值 ecd
```

**重要**：LKMotorInit中有一处逻辑需要注意（第56行有重复malloc，属于框架小bug）：
```c
LKMotorInstance *motor = (LKMotorInstance *)malloc(sizeof(LKMotorInstance));
motor = (LKMotorInstance *)malloc(sizeof(LKMotorInstance));  // 第二次,造成内存泄漏
memset(motor, 0, sizeof(LKMotorInstance));
```
虽然有内存泄漏但不影响功能。如果介意，可删除重复的那一行。

### 2.3 CAN2总线上共存两种协议

| 时间片 | CAN ID | 内容 | 发送方 |
|--------|--------|------|--------|
| 控制发送 | 0x200 | M3508电流指令(0~2字节有效) | STM32 |
| 控制发送 | 0x280 | LK9015电流指令(0~3字节有效) | STM32 |
| 反馈接收 | 0x201 | 左舵向电机反馈 | M3508电调 |
| 反馈接收 | 0x202 | 右舵向电机反馈 | M3508电调 |
| 反馈接收 | 0x141 | 左轮向电机反馈 | LK9015 |
| 反馈接收 | 0x142 | 右轮向电机反馈 | LK9015 |

## 三、初始化四个电机的完整代码

### 3.1 创建验证模块

建议在 `application/` 目录下创建 `motor_test.c/h` 用于电机验证：

```c
// application/motor_test.h
#ifndef MOTOR_TEST_H
#define MOTOR_TEST_H

#include "motor_def.h"
#include "dji_motor.h"
#include "LK9025.h"

// 四个电机实例指针(全局保存,方便验证时访问)
extern DJIMotorInstance *motor_steer_left;   // 左舵向 M3508, ID=1
extern DJIMotorInstance *motor_steer_right;  // 右舵向 M3508, ID=2
extern LKMotorInstance  *motor_wheel_left;   // 左轮向 LK9015, ID=1
extern LKMotorInstance  *motor_wheel_right;  // 右轮向 LK9015, ID=2

void MotorTestInit(void);    // 初始化四个电机
void MotorTestRun(void);     // 验证用:发送简单控制指令
void MotorTestPrintStatus(void); // 打印四个电机状态(用于调试)

#endif // MOTOR_TEST_H
```

### 3.2 初始化代码

```c
// application/motor_test.c
#include "motor_test.h"
#include "bsp_log.h"

DJIMotorInstance *motor_steer_left;   // 左舵向 M3508, ID=1
DJIMotorInstance *motor_steer_right;  // 右舵向 M3508, ID=2
LKMotorInstance  *motor_wheel_left;   // 左轮向 LK9015, ID=1
LKMotorInstance  *motor_wheel_right;  // 右轮向 LK9015, ID=2

/**
 * @brief 初始化一个DJI电机的通用配置
 * @param tx_id 电调ID(1~4)
 */
static Motor_Init_Config_s make_dji_config(uint8_t tx_id) {
    Motor_Init_Config_s conf = {0};

    // CAN配置: 使用CAN2, tx_id=电调闪烁次数
    conf.can_init_config.can_handle = &hcan2;
    conf.can_init_config.tx_id = tx_id;

    // 电机类型: M3508
    conf.motor_type = M3508;

    // 控制设置: 速度环 + 电流环, 不反转, 不使用其他反馈源
    conf.controller_setting_init_config.outer_loop_type = SPEED_LOOP;
    conf.controller_setting_init_config.close_loop_type = SPEED_LOOP | CURRENT_LOOP;
    conf.controller_setting_init_config.angle_feedback_source = MOTOR_FEED;
    conf.controller_setting_init_config.speed_feedback_source = MOTOR_FEED;
    conf.controller_setting_init_config.motor_reverse_flag = MOTOR_DIRECTION_NORMAL;
    conf.controller_setting_init_config.feedback_reverse_flag = 0;
    conf.controller_setting_init_config.feedforward_flag = 0;

    // PID参数(保守值,验证用,不追求性能)
    conf.controller_param_init_config.speed_PID.Kp = 5.0f;
    conf.controller_param_init_config.speed_PID.Ki = 0.1f;
    conf.controller_param_init_config.speed_PID.Kd = 0.0f;
    conf.controller_param_init_config.speed_PID.MaxOut = 8000.0f;   // 限制最大输出(给电流环的参考)
    conf.controller_param_init_config.speed_PID.IntegralLimit = 2000.0f;
    conf.controller_param_init_config.speed_PID.DeadBand = 1.0f;
    conf.controller_param_init_config.speed_PID.Improve = PID_Trapezoid_Intergral | PID_Integral_Limit | PID_Derivative_On_Measurement;

    conf.controller_param_init_config.current_PID.Kp = 1.0f;
    conf.controller_param_init_config.current_PID.Ki = 0.0f;
    conf.controller_param_init_config.current_PID.Kd = 0.0f;
    conf.controller_param_init_config.current_PID.MaxOut = 16000.0f; // M3508最大16384
    conf.controller_param_init_config.current_PID.Improve = PID_Trapezoid_Intergral | PID_Integral_Limit | PID_Derivative_On_Measurement;

    // 位置环参数(虽然不用,但DJIMotorInit仍然会初始化它,设为安全值)
    conf.controller_param_init_config.angle_PID.Kp = 0.0f;
    conf.controller_param_init_config.angle_PID.Ki = 0.0f;
    conf.controller_param_init_config.angle_PID.Kd = 0.0f;
    conf.controller_param_init_config.angle_PID.MaxOut = 0.0f;

    conf.controller_param_init_config.other_angle_feedback_ptr = NULL;
    conf.controller_param_init_config.other_speed_feedback_ptr = NULL;
    conf.controller_param_init_config.speed_feedforward_ptr = NULL;
    conf.controller_param_init_config.current_feedforward_ptr = NULL;

    return conf;
}

/**
 * @brief 初始化一个LK电机的通用配置
 * @param tx_id 电机ID(1~4)
 */
static Motor_Init_Config_s make_lk_config(uint8_t tx_id) {
    Motor_Init_Config_s conf = {0};

    conf.can_init_config.can_handle = &hcan2;
    conf.can_init_config.tx_id = tx_id;   // 1~4

    // 注: LK9015的类型枚举... 框架里只有M3508/M2006等DJI型号
    // LKMotorInit内部实际上不检查motor_type, 但为了避免意外给一个安全值
    conf.motor_type = M3508; // LK电机的初始化函数不真正使用这个字段

    // 控制设置: 只用电流环(开环测试最简单,先验证电机能转)
    // 如果用速度环需注意: LK代码中speed_aps的反馈源处理有bug,建议先用电流环
    conf.controller_setting_init_config.outer_loop_type = CURRENT_LOOP;
    conf.controller_setting_init_config.close_loop_type = CURRENT_LOOP;
    conf.controller_setting_init_config.angle_feedback_source = MOTOR_FEED;
    conf.controller_setting_init_config.speed_feedback_source = MOTOR_FEED;
    conf.controller_setting_init_config.motor_reverse_flag = MOTOR_DIRECTION_NORMAL;
    conf.controller_setting_init_config.feedback_reverse_flag = 0;
    conf.controller_setting_init_config.feedforward_flag = 0;

    // PID参数: LK9015只用电流环(给固定电流值)
    conf.controller_param_init_config.current_PID.Kp = 1.0f;
    conf.controller_param_init_config.current_PID.Ki = 0.0f;
    conf.controller_param_init_config.current_PID.Kd = 0.0f;
    conf.controller_param_init_config.current_PID.MaxOut = 2000.0f; // LK最大±2000
    conf.controller_param_init_config.current_PID.Improve = PID_Integral_Limit;

    // 其他环给安全默认值
    conf.controller_param_init_config.speed_PID.Kp = 0.0f;
    conf.controller_param_init_config.speed_PID.Ki = 0.0f;
    conf.controller_param_init_config.speed_PID.Kd = 0.0f;
    conf.controller_param_init_config.speed_PID.MaxOut = 0.0f;

    conf.controller_param_init_config.angle_PID.Kp = 0.0f;
    conf.controller_param_init_config.angle_PID.Ki = 0.0f;
    conf.controller_param_init_config.angle_PID.Kd = 0.0f;
    conf.controller_param_init_config.angle_PID.MaxOut = 0.0f;

    conf.controller_param_init_config.other_angle_feedback_ptr = NULL;
    conf.controller_param_init_config.other_speed_feedback_ptr = NULL;
    conf.controller_param_init_config.speed_feedforward_ptr = NULL;
    conf.controller_param_init_config.current_feedforward_ptr = NULL;

    return conf;
}

/**
 * @brief 初始化四个电机
 */
void MotorTestInit(void) {
    Motor_Init_Config_s conf;

    // --- 左舵向 M3508, tx_id=1 ---
    conf = make_dji_config(1);
    motor_steer_left = DJIMotorInit(&conf);
    LOGINFO("[MotorTest] 左舵向M3508 ID=1 已初始化, 实例=%p", motor_steer_left);

    // --- 右舵向 M3508, tx_id=2 ---
    conf = make_dji_config(2);
    motor_steer_right = DJIMotorInit(&conf);
    LOGINFO("[MotorTest] 右舵向M3508 ID=2 已初始化, 实例=%p", motor_steer_right);

    // --- 左轮向 LK9015, tx_id=1 ---
    conf = make_lk_config(1);
    motor_wheel_left = LKMotorInit(&conf);
    LOGINFO("[MotorTest] 左轮向LK9015 ID=1 已初始化, 实例=%p", motor_wheel_left);

    // --- 右轮向 LK9015, tx_id=2 ---
    conf = make_lk_config(2);
    motor_wheel_right = LKMotorInit(&conf);
    LOGINFO("[MotorTest] 右轮向LK9015 ID=2 已初始化, 实例=%p", motor_wheel_right);

    LOGINFO("[MotorTest] 四个电机初始化完成");
}

/**
 * @brief 验证测试运行: 给每个电机一个小的参考值让它缓慢转动
 * 建议放在某个RTOS任务中, 或者在RobotTask中临时调用
 */
void MotorTestRun(void) {
    // 测试M3508: 给一个小的速度参考(单位:度/秒)
    // 注意: M3508的外环是SPEED_LOOP, 参考值是度/秒
    DJIMotorSetRef(motor_steer_left,  50.0f);  // 50度/秒 = 约8.3rpm, 缓慢转动
    DJIMotorSetRef(motor_steer_right, 50.0f);

    // 测试LK9015: 外环是CURRENT_LOOP, 参考值直接作为PID输出
    // LK最大电流值范围是-2000~2000(约-10A~+10A),给100相当于很小的电流
    LKMotorSetRef(motor_wheel_left,  100.0f);
    LKMotorSetRef(motor_wheel_right, 100.0f);
}

/**
 * @brief 打印四个电机的反馈状态(方便用SWV/J-Link RTT查看)
 */
void MotorTestPrintStatus(void) {
    // DJI M3508
    LOGINFO("[M3508-L] ecd=%d, angle=%.1fdeg, speed=%.1fdeg/s, current=%d, online=%d",
            motor_steer_left->measure.ecd,
            motor_steer_left->measure.angle_single_round,
            motor_steer_left->measure.speed_aps,
            motor_steer_left->measure.real_current,
            DaemonIsOnline(motor_steer_left->daemon));

    LOGINFO("[M3508-R] ecd=%d, angle=%.1fdeg, speed=%.1fdeg/s, current=%d, online=%d",
            motor_steer_right->measure.ecd,
            motor_steer_right->measure.angle_single_round,
            motor_steer_right->measure.speed_aps,
            motor_steer_right->measure.real_current,
            DaemonIsOnline(motor_steer_right->daemon));

    // LK9015
    LOGINFO("[LK9015-L] ecd=%d, angle=%.1fdeg, speed=%.3frad/s, current=%d, online=%d",
            motor_wheel_left->measure.ecd,
            motor_wheel_left->measure.angle_single_round,
            motor_wheel_left->measure.speed_rads,
            motor_wheel_left->measure.real_current,
            LKMotorIsOnline(motor_wheel_left));

    LOGINFO("[LK9015-R] ecd=%d, angle=%.1fdeg, speed=%.3frad/s, current=%d, online=%d",
            motor_wheel_right->measure.ecd,
            motor_wheel_right->measure.angle_single_round,
            motor_wheel_right->measure.speed_rads,
            motor_wheel_right->measure.real_current,
            LKMotorIsOnline(motor_wheel_right));
}
```

### 3.3 集成到框架

修改 `application/robot.c` 的 `RobotInit()` 调用初始化：

```c
// 在 application/robot.c 中
#include "motor_test.h"   // 添加

void RobotInit() {
    __disable_irq();
    BSPInit();

    // 这里原本可能有RobotCMDInit/ChassisInit等,保持不变
    // ...

    // ===== 添加: 初始化四个电机用于验证 =====
    MotorTestInit();

    OSTaskInit();
    __enable_irq();
}

// 然后在某个RTOS任务中(比如RobotTask)添加验证调用
void RobotTask() {
    // ... 原有的任务内容

    // ===== 电机验证部分 =====
    static uint32_t test_cnt = 0;
    MotorTestRun();               // 设置控制指令

    // 每隔一定时间打印一次状态(约1秒)
    if (++test_cnt % 1000 == 0) {
        MotorTestPrintStatus();
    }
}
```

## 四、验证流程（一步步做）

### Step 1：确认硬件连接

- [ ] CAN2收发器已焊接，CANH/CANL已连接到四个电机
- [ ] M3508电调ID：一个设为1（闪烁1次），一个设为2（闪烁2次）
- [ ] LK9015电调ID：用LK上位机设置ID=1和ID=2
- [ ] 电机供电电压正常（24V）
- [ ] STM32供电正常，USB/ST-Link已连接

### Step 2：编译配置

- [ ] `robot_def.h` 中 `ONE_BOARD` 被定义（单板模式验证最简单）
- [ ] 确保 `modules/motor/DJImotor/` 和 `modules/motor/LKmotor/` 被加入编译路径
- [ ] `motor_task.c` 中 `LKMotorControl()` 被调用（已有）

### Step 3：下载程序并观察日志

1. 下载程序，打开串口/J-Link RTT查看日志
2. 正常应该看到：
   ```
   [MotorTest] 左舵向M3508 ID=1 已初始化
   [MotorTest] 右舵向M3508 ID=2 已初始化
   [MotorTest] 左轮向LK9015 ID=1 已初始化
   [MotorTest] 右轮向LK9015 ID=2 已初始化
   [MotorTest] 四个电机初始化完成
   ```

## 五、关键变量说明与联系

### 5.1 DJI电机实例（DJIMotorInstance）

```
DJIMotorInstance
├── measure (DJI_Motor_Measure_s)        ← 【反馈数据区,从CAN报文解析来】
│   ├── ecd                ← uint16_t, 编码器0~8191
│   ├── angle_single_round ← float, 单圈角度0~360
│   ├── speed_aps          ← float, 角速度度/秒
│   ├── real_current       ← int16_t, 实际电流
│   ├── temperature        ← uint8_t, 温度
│   ├── total_angle        ← float, 多圈累计角度
│   └── total_round        ← int32_t, 圈数
├── motor_settings (Motor_Control_Setting_s) ← 【控制配置,初始化设置】
│   ├── outer_loop_type    ← SPEED_LOOP (外环类型)
│   ├── close_loop_type    ← SPEED_LOOP|CURRENT_LOOP (启用哪些环)
│   ├── motor_reverse_flag ← 方向是否反转
│   ├── angle/speed_feedback_source ← 反馈源(编码器/外部)
│   └── feedforward_flag   ← 前馈标志
├── motor_controller (Motor_Controller_s)  ← 【PID控制器】
│   ├── current_PID        ← 电流环PID
│   ├── speed_PID          ← 速度环PID
│   ├── angle_PID          ← 位置环PID
│   └── pid_ref            ← float, 外环参考输入 ←【DJIMotorSetRef设置这个】
├── motor_can_instance (CANInstance*)     ← 【CAN总线实例】
│   ├── rx_id              ← 0x201或0x202 (接收)
│   ├── tx_id              ← 0x200 (发送,分组管理)
│   └── can_handle         ← &hcan2
├── sender_group           ← 发送分组(=4, CAN2上的0x200组)
├── message_num            ← 在分组中的位置(0或1)
├── stop_flag              ← MOTOR_STOP / MOTOR_ENALBED
├── daemon (DaemonInstance*) ← 看门狗实例
└── dt, feed_cnt           ← 时间戳/计数
```

**数据流方向**：
```
CAN接收 → DecodeDJIMotor() → measure区更新 → PID计算 → tx_buff填充 → CAN发送
```

### 5.2 LK电机实例（LKMotorInstance）

```
LKMotorInstance
├── measure (LKMotor_Measure_t)           ← 【反馈数据区】
│   ├── ecd                ← uint16_t, 编码器0~65535
│   ├── angle_single_round ← float, 单圈角度
│   ├── speed_rads         ← float, 角速度(弧度/秒)!!!
│   ├── real_current       ← int16_t, 实际电流
│   ├── temperature        ← uint8_t, 温度
│   ├── total_angle/total_round ← 累计
│   └── feed_dwt_cnt       ← DWT时间戳
├── motor_settings         ← 控制配置(同DJI)
├── current_PID/speed_PID/angle_PID ← 三环PID
├── pid_ref                ← float, 参考输入 ←【LKMotorSetRef设置这个】
├── stop_flag              ← 启停标志
├── motor_can_ins          ← CAN实例
│   ├── rx_id = 0x141/0x142
│   └── tx_id = 0x280/0x281 (第一个电机的0x280实际被使用)
└── daemon                 ← 看门狗
```

**LK与DJI的主要区别**：
1. 编码器分辨率：M3508=8192(13bit), LK9015=65536(16bit)
2. 速度单位：M3508=度/秒, LK9015=弧度/秒
3. 发送方式：M3508用sender_assignment分组, LK用第一个注册的电机的sender_instance
4. 电流范围：M3508±16384(±20A), LK±2000(±10A)

### 5.3 控制回路数据流

**DJI M3508（速度环+电流环）**：
```
DJIMotorSetRef(motor, 50.0f)
       ↓ 设置 pid_ref = 50 (度/秒)
DJIMotorControl() 被调用 (500Hz)
├─ 速度环: PIDCalculate(speed_PID, measure.speed_aps, 50) → 输出参考电流
├─ 电流环: PIDCalculate(current_PID, measure.real_current, 参考电流) → 输出set
└─ set(int16_t) 填入CAN发送buffer → CANTransmit(0x200)
```

**LK9015（仅电流环，简化测试）**：
```
LKMotorSetRef(motor, 100.0f)
       ↓ 设置 pid_ref = 100
LKMotorControl() 被调用
└─ 电流环: PIDCalculate(current_PID, measure.real_current, 100) → 输出set
   set填入sender_instance->tx_buff[0~1]或[2~3] → CANTransmit(0x280)
```

## 六、常见问题排查

### 问题1：DJI电机 online=0，不反馈

**现象**：`DaemonIsOnline(motor->daemon)` 返回0，`measure.ecd` 恒为0

**排查**：
1. CAN2初始化是否成功？`MX_CAN2_Init()` 被调用了吗？在 `main.c` 中检查
2. 电调是否上电？电调指示灯是否闪烁？
3. ID是否匹配？M3508电调闪烁次数是否为1、2？
4. CANH/CANL是否反接？
5. 终端电阻是否安装？(两端120Ω)

### 问题2：LK电机 online=0，不反馈

**现象**：`LKMotorIsOnline()` 返回0

**排查**：
1. LK电调的ID是否通过上位机正确设置为1、2？
2. 协议是否匹配？LK9015的反馈ID是0x141、0x142，需要电调支持这个协议
3. LK9015需要上电后约2秒才开始发送反馈

### 问题3：电机有反馈但不转动

**现象**：`measure.ecd` 变化，但肉眼观察电机静止

**排查**：
1. `pid_ref` 是否被正确设置？在调试器中监视 `motor_controller.pid_ref`
2. `stop_flag` 是否为 `MOTOR_ENALBED`(1)？
3. PID输出 `set` 是否为非零值？
4. 电机是否被堵转？给更大的电流试试（LK:500→2000）

### 问题4：电机抖动剧烈

**现象**：电机来回抖动，电流较大

**排查**：
1. 减小 `speed_PID.Kp`（从5.0降到0.5试试）
2. 检查反馈值是否真实反映了电机状态（无噪声）
3. `speed_PID.MaxOut` 是否过大？导致输出饱和

### 问题5：CAN总线拥堵/仲裁失败

**现象**：日志中有CAN error，或部分电机偶尔online=0

**排查**：
1. 降低 `MotorControlTask()` 的执行频率（500Hz→100Hz）
2. 确认每个电机的反馈频率不超过500Hz（LK电机可通过上位机设置）
3. 检查是否有其他模块也在高频使用CAN2

### 问题6：DJI和LK共用CAN2导致ID冲突

**说明**：理论上不会冲突：
- DJI M3508使用0x200（发送）、0x201/0x202（接收）
- LK9015使用0x280（发送）、0x141/0x142（接收）

**但需确认**：LK电调是否还发送其他CAN ID的报文？有些LK电调的心跳/参数帧ID可能与DJI冲突。
